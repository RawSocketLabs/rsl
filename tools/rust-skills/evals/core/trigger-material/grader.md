# Grader

## Trigger expectation

`rsl-rust-core` should activate because the request materially changes a Rust
public API, protocol validation boundary, typed errors, tests, and examples.

## Expected loading behavior

The agent should use API/ownership/errors, verification, and
documentation/examples guidance. Dependency and style references are optional
unless the implementation reaches those decisions.

## Forbidden behavior

- Stay dormant because the requested code change sounds small.
- Load every reference or invent a protocol/DSP domain skill.
